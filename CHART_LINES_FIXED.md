## 🔧 **1️⃣ 세력 평단 - 개선됨 ✅**

### **변경 전 (❌ 부정확)**
```javascript
const forceAveragePrice = (baseCandle.high + baseCandle.low + baseCandle.close) / 3;
// 문제: 고가를 포함 → 세력이 확보한 가격 아님
```

### **변경 후 (✅ 정확)**
```javascript
const forceAveragePrice = (baseCandle.low + baseCandle.close) / 2;
// 개선: 저가와 종가의 평균
// 이유: 세력은 "저가"에서 물량 확보, 고가는 매물대 테스트일 뿐
```

### **의미 변화**
- 이전: 임의의 평균값
- **이후**: 세력의 실제 확보 평단에 더 가까움

**차트 범례:**
```
🔵 세력 평단 (저+종)/2
```

## 🔧 **2️⃣ 매수라인 - 명확화 ✅**

### **변경 전 (⚠️ 복잡한 로직)**
```javascript
const buyLine =
  latest.close > recentHigh && latest.volumeRatio >= 1.05
    ? Math.max(latest.close * 0.985, pullbackBuyLine)
    : pullbackDepth <= 0.055
      ? pullbackBuyLine
      : Math.min(breakoutBuyLine, latest.close * 1.015);
// 문제: 복잡하고 단테기법 원칙 불명확
```

### **변경 후 (✅ 명확한 3가지 기준)**

#### **단테 눌림목 진입의 3가지 기준:**
```javascript
// 1️⃣ 정상 눌림 (3~5.5% 고점 대비 하락)
const normalPullbackBuyLine = recentHigh * (1 - Math.min(Math.max(0.035, pullbackDepth), 0.055));

// 2️⃣ 3일선 지지
const threeDayLineBuyLine = sma3;

// 3️⃣ 손절선 위쪽 (리스크 관리)
const riskLineBuyLine = riskLine * 1.015;

// 최종 = 세 기준 중 가장 높은 지점 (충분한 리스크 리워드)
const pullbackBuyLine = Math.max(normalPullbackBuyLine, threeDayLineBuyLine, riskLineBuyLine);
```

#### **상황별 매수라인:**
```javascript
// 상황 1: 돌파 후 정상 눌림 → 돌파 기준으로 추격
if (latest.close > recentHigh && latest.volumeRatio >= 1.05 && pullbackDepth <= 0.055)
  → buyLine = breakoutBuyLine (돌파 추격)

// 상황 2: 정상 눌림 중, 3일선 위 → 눌림목 진입
else if (pullbackDepth > 0 && pullbackDepth <= 0.055 && latest.close >= sma3)
  → buyLine = pullbackBuyLine (눌림목 진입)

// 상황 3: 신호 미약 → 대기
else
  → buyLine = recentHigh * 1.001 (신호 대기)
```

### **의미 변화**
- 이전: 애매한 "매수라인"
- **이후**: 
  - `돌파 추격` - 저항선 돌파 후 거래량 + 정상 눌림
  - `눌림목 진입` - 상승 초입 후 3~5.5% 조정
  - `신호 대기` - 조건 미충족

**차트 범례:**
```
🔵 눌림목 진입 / 돌파 추격 / 신호 대기
```
(상황에 따라 동적으로 변경)

## 🔧 **4️⃣ 손절 기준 - 유지 ✅**

### **이미 완벽 구현 (최근 개선)**
```javascript
const criteriaLow = baseCandle.low;           // 1️⃣ 기준봉 저가
const criteriaSupport = supportLine;          // 2️⃣ 지지선
const riskLine = Math.min(criteriaLow, criteriaSupport);
// 손절 = 둘 중 낮은 값
```

### **의미**
- 단테기법의 손절 원칙 100% 준수
- 기준봉저가 or 지지선 중 하나 이탈 → 즉시 손절
