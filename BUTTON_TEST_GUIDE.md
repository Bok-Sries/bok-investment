## 🧪 **버튼 작동 테스트 방법**

### **Step 1: 개발자 도구 열기**
```
F12 또는 우클릭 → "검사" → "Console" 탭
```

### **Step 2: 콘솔 메시지 확인**
```
앱 로드 시 다음 메시지 확인:

✅ 시작 버튼 리스너 등록됨
✅ 알림 버튼 리스너 등록됨
✅ 중지 버튼 리스너 등록됨
✅ 삭제 버튼 리스너 등록됨
```

### **Step 3: 버튼 클릭 테스트**

#### **3-1. 종목 선택 (필수)**
```
1. 드롭다운에서 종목 선택
2. 예: 005930 (삼성전자)
```

#### **3-2. 알림 활성화 클릭**
```
1. "🔊 알림 활성화" 버튼 클릭
2. 콘솔에 나타날 메시지:
   └─ 🔊 알림 권한 요청
3. 브라우저 알림 권한 다이얼로그 나타남
4. "허용" 선택
```

#### **3-3. 시작 버튼 클릭**
```
1. "▶️ 시작" 버튼 클릭
2. 콘솔에 나타날 메시지:
   └─ 🔔 모니터링 시작: 005930
   └─ ✅ 005930 모니터링 시작 (5분 간격)
3. 버튼 변경:
   ├─ "▶️ 시작" → 숨김
   └─ "⏹️ 중지" → 표시됨
4. 상태 표시:
   └─ "모니터링 중..." (초록색)
```

#### **3-4. 알림 히스토리 확인**
```
1. 시작 후 5분 대기
2. 신호 변화 시 자동 알림:
   └─ 🔔 데스크톱 알림 표시
   └─ 📝 알림 히스토리 추가
3. 알림 히스토리 패널:
   ├─ 최근 알림 표시됨
   ├─ 각 알림의 시간 표시됨
   └─ "🗑️ 모두 삭제" 버튼 활성화됨
```

#### **3-5. 중지 버튼 클릭**
```
1. "⏹️ 중지" 버튼 클릭
2. 콘솔에 나타날 메시지:
   └─ 🛑 모니터링 중지: 005930
3. 버튼 변경:
   ├─ "⏹️ 중지" → 숨김
   └─ "▶️ 시작" → 표시됨
4. 상태 표시:
   └─ "대기" (회색)
```

## 🐛 **문제 해결**

### **Q1: "버튼을 찾을 수 없습니다" 에러**
```
해결 방법:
1. F12 Console에서 직접 확인:
   document.querySelector("#startMonitoringBtn")
   → element가 나타나면 정상
   → null이 나타나면 HTML ID 확인

2. HTML에서 버튼 ID 재확인:
   id="startMonitoringBtn"
   id="requestNotificationBtn"
```

### **Q2: 버튼 클릭해도 반응 없음**
```
해결 방법:
1. 콘솔에서 설정 확인:
   alertStorage.monitoringActive
   
2. 종목이 선택되었는지 확인:
   elements.ticker.value
   → 값이 없으면 종목 선택 필요
```

### **Q3: 모니터링이 시작되지 않음**
```
해결 방법:
1. 콘솔 에러 확인
2. 함수 직접 테스트:
   startSignalMonitoring('005930')
   
3. 5분 간격 확인:
   console.log(alertStorage.monitoringIntervals)
```

### **Q4: 알림이 나타나지 않음**
```
해결 방법:
1. 알림 권한 확인:
   Notification.permission
   → "granted" 확인
   
2. "알림 활성화" 다시 클릭
   
3. 신호 변화 확인:
   5분 후 신호 점수 변화 시 알림
```
